## System Features

- **Expert Discovery Interface:** Responsive grid layout for expert discovery, including text search, category filtering, and performance rating metrics.
- **Regional Localization:** Localization tailored for the Indian region, incorporating Indian Rupee (`₹`) currency formatting, IST-anchored time scheduling, and strict 10-digit mobile number validation with a `+91` prefix.
- **Real-Time Availability Synchronization:** Live time-slot status synchronization utilizing Socket.io rooms to broadcast booking state changes instantly.
- **Concurrency Control:** Two-layer concurrency defense mechanisms utilizing pre-check queries and compound unique partial database indices to prevent duplicate bookings.
- **Multi-Document Consistency:** ACID-compliant MongoDB transactions applied across distributed collections to prevent partial-data corruption during complex mutations.
- **Automated Image Fallbacks:** Autogenerated, personalized, initials-based user avatars integrated via `ui-avatars.com`.
- **Authentication and RBAC:** JWT-based credential verification flows, state persistence, and Role-Based Access Control (RBAC) boundaries for `Client`, `Expert`, and `Admin` entities.
- **Role-Specific Dashboards:** Centralized administration consoles enabling system management, schedule configuration, business analytics, and slot availability blocking.
- **Cancellation and Penalty Enforcement:** Enforcement of a 2-hour IST cancellation window, including automated tracking and account suspension for users exceeding late-cancellation thresholds.
- **Peer-to-Peer Feedback System:** Double-sided review mechanism facilitating mutual rating inputs between Clients and Experts post-session.
- **Automated Notifications:** Persistent asynchronous scheduler utilizing Agenda.js to dispatch booking confirmations, cancellations, and pre-session reminders via Nodemailer and Twilio.
- **Credential Recovery:** Cryptographic token-based password recovery flows enforcing 10-minute expiry constraints.
- **Real-Time Messaging:** Private, booking-scoped WebSocket chat messaging between Clients and Experts.
- **Payment Gateway Integration:** Razorpay checkout integration featuring client-side signature verification, transactional idempotency, Two-Phase Commit boundaries, and automated slot-release upon payment failure.
- **API Security Hardening:** Implementation of Zod schema validation, ReDoS sanitization, IP rate limiting, DTO serialization, and Express 5.0 NoSQL sanitization wrappers.
- **User Interface and Pagination:** Custom React dialog components for user prompts and strict server-side query pagination to optimize large dataset retrieval.
- **Agentic AI Integration:** Core architectural integration with the Model Context Protocol (MCP) SDK to facilitate advanced AI-agent interactions and localized querying.

## 📂 Architecture & Project Structure

The project follows a standard decoupled Monorepo pattern with strict **3-Tier Clean Architecture** on the backend.

```text
Real-Time-Expert-Session-Booking-System/
├── backend/                  # Node.js, Express, MongoDB API
│   ├── src/
│   │   ├── config/           # Database and third-party configuration
│   │   ├── controllers/      # HTTP request parsing and response formatting
│   │   ├── middleware/       # Auth, Uploads, Error Handling, and Zod Validations
│   │   ├── models/           # Mongoose Schemas (User, Booking, Expert, etc.)
│   │   ├── repositories/     # Data Access Layer (DAL) for database interactions
│   │   ├── routes/           # API Endpoint mounts
│   │   ├── seeds/            # Database seeder scripts and utilities
│   │   ├── services/         # Core Business Logic and Transaction Boundaries
│   │   └── __tests__/        # Colocated Unit test suites (White-box testing)
│   ├── tests/                # Root Integration/E2E test suites (Black-box testing)
│   ├── .env.example          # Environment variable template
│   └── package.json          # Backend dependencies
├── frontend/                 # React + Vite Client
│   ├── src/
│   │   ├── components/       # Reusable UI components (Navbar, Cards, Modals)
│   │   ├── context/          # React Context providers (Auth, Notifications)
│   │   ├── pages/            # Routable application views (Dashboards, Booking, Chat)
│   │   └── services/         # Axios API interceptors and Socket connections
│   └── package.json          # Frontend dependencies
├── docs/                     # Engineering documentation and specifications
├── graphify-out/             # Generated Codebase Knowledge Graph & AST outputs
├── .github/                  # CI/CD pipelines and automated dependency management
├── start.sh                  # Application startup utility script
└── README.md                 # Project documentation
```

## ⚙️ Getting Started

### Prerequisites
- **Node.js:** v18 or higher.
- **MongoDB:** A replica set configuration (for local standalone) or a MongoDB Atlas cluster URI (to support ACID transactions).
- **Redis:** (Optional for local development, required for multi-instance scaling).

### Quick Start
You can boot both the frontend and backend simultaneously using the provided auto-start script:
```bash
# Grants execution permissions and starts the dev servers
chmod +x start.sh
./start.sh
```
*(Proactively clears legacy ghost processes on ports 5000 and 5173 to ensure a clean startup).*

### Manual Setup

#### 1. Backend Setup
```bash
cd backend
npm install
```
Create a `.env` file in the `backend/` directory:
```env
PORT=5000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret_key
RAZORPAY_KEY_ID=your_razorpay_key
RAZORPAY_KEY_SECRET=your_razorpay_secret
RAZORPAY_WEBHOOK_SECRET=your_webhook_secret
```
*(Optional) Seed the database:*
```bash
node src/seeds/userSeeder.js
node src/seeds/expertSeeder.js
```
Start the backend server:
```bash
npm run dev
```

#### 2. Frontend Setup
```bash
cd frontend
npm install
npm run dev
```

#### 3. Running Tests
The backend strictly separates unit and integration tests. Run them using:
```bash
cd backend
npm run test:unit         # Executes fast, isolated unit tests
npm run test:integration  # Executes database-connected integration suites
npm test                  # Executes the full V&V test pipeline
```

## 🧪 Testing & Quality Assurance

### Technology Matrix

| Testing Level | Technology | Target |
|---|---|---|
| **Unit Testing** | Jest (Backend), Vitest (Frontend) | Component Logic, Backend Services & Business Logic |
| **Integration Testing** | Jest + SuperTest | API Endpoints & Transaction Boundaries |
| **E2E / System Testing** | Playwright | Full User Journeys |
| **Performance Testing** | k6 | Load and Stress thresholds |
| **Static Code Analysis** | ESLint | Code Quality and Formatting |

### Formal Verification & Validation (V&V)
SkillSync strictly adheres to international software engineering documentation standards:
- **ISO/IEC/IEEE 29119-3:** Automated tests are documented using standardized Test Execution Logs and Test Completion Reports.
- **IEEE Std 1012:** Codebase modifications are gated by formal V&V execution reports and anomaly tracking.
- **Integration Test Sandbox:** Critical API boundaries are subjected to automated `jest` and `supertest` verification using an isolated `mongodb-memory-server` database, ensuring 100% test pass rates before deployment.
- **Engineering Principle Compliance:** The codebase is systematically audited against a 270-principle framework managing technical debt and Engineering Maturity.

### Continuous Integration & Deployment (CI/CD)
- **GitHub Actions:** Automated test pipelines execute on all Pull Requests and merges to the main branch.
- **Dependabot:** Proactive tracking and automated merging of security patches and dependency bumps.

### Testing Architecture Boundaries
To enforce maximum isolation and test reliability, the backend adheres to a strict physical boundary split:
- **Unit Tests (`src/__tests__/unit/`):** Colocated with the source files they test. They execute rapidly and mock external database/service dependencies, verifying granular logic (White-box testing).
- **Integration Tests (`tests/integration/`):** Situated at the project root outside `src/`. They load the fully compiled Express application and hit endpoints via HTTP requests, verifying end-to-end database transactions and API responses (Black-box testing).

## 📜 Coding Guidelines

- **Frontend:** Built on Vite. Use `npm run dev` for development, `npm run build` for production builds, and `npm run lint` for code quality checks.
- **Backend Architecture:** Adheres strictly to a **3-Tier Clean Architecture**. Responsibilities are separated across Routes, Controllers, Services (Domain Logic), Repositories (DAL), and Models. **Never** place business logic directly in the Controller.
- **Real-Time Communication:** Relies heavily on accurate Room management in Socket.io to optimize network bandwidth and ensure targeted broadcasts.
