## 🚀 Features

- **Landing Page & Expert Discovery:** Responsive grid of experts with search, category filtering, and rating metrics.
- **India-Specific Localization:** Fully localized context including Indian Rupee (`₹`), IST-anchored time scheduling, and strict 10-digit mobile number input with `+91` prefixing.
- **Real-Time Availability:** Live time slot status synchronization via Socket.io rooms to reflect bookings instantly.
- **Conflict-Free Booking:** Two-layer concurrency defense (pre-check query and compound unique partial database index).
- **Robust Multi-Document Consistency:** ACID-compliant MongoDB transactions across distributed collections to eliminate partial-data corruption.
- **Smart Image Fallbacks:** Autogenerated, personalized, initials-based avatars powered by `ui-avatars.com`.
- **JWT Authentication & RBAC:** Secure sign-up/login flows, state persistence, authorization checks for `Client`, `Expert`, and `Admin` roles.
- **Admin & Expert Dashboards:** Centralized consoles for system administration, schedule management, business analytics, and slot blocking.
- **Late Cancellation & Strike System:** Enforces a 2-hour IST cancellation window and tracks/suspends users with excessive late cancellations.
- **Two-Sided P2P Feedback System:** Implements a double-sided trust network for mutual Client/Expert ratings post-session.
- **Automated Email & SMS Reminders:** Persistent scheduler using Agenda.js for booking confirmations, cancellations, and pre-session reminders (Nodemailer/Twilio).
- **Password Recovery & Auto-Login:** Secure Forgot Password flows with crypto tokens and 10-minute expiries.
- **Real-Time Messaging:** Private, booking-bounded chat messaging between Clients and Experts.
- **Payment Gateway Integration:** Razorpay checkout with client-side signature verification, idempotency, **Two-Phase Commit boundaries**, and automatic slot-release on failure.
- **API Security Hardening:** **Zod schema validation**, ReDoS sanitization, IP rate limiting, DTO serialization, Express 5.0 NoSQL sanitization wrappers.
- **Custom UX & Pagination:** Glassmorphic React dialogs replacing native browser prompts, and strict query pagination.

## 🛠️ Tech Stack

### Frontend Core
- **React (v19.2) & Vite (v8.0):** Component-driven interface library with an ultra-fast compilation bundler.
- **Tailwind CSS (v3.4):** Utility-first styling framework with custom glassmorphism components.
- **React Router DOM (v7.15) & Lucide React (v1.14):** Declarative routing and premium SVG iconography.
- **Axios (v1.16) & Socket.io Client (v4.8):** Promise-based HTTP and real-time persistent WebSocket clients.

### Backend Core
- **Node.js (v18+) & Express (v5.2):** Robust Javascript runtime and minimalist server framework.
- **Mongoose (v9.6) & MongoDB Atlas:** Schema-based ODM layer enforcing ACID transactions (`session.withTransaction()`).
- **Socket.io (v4.8) & Redis (v6.0):** WebSockets engine with `@socket.io/redis-adapter` for horizontally scaling real-time state.
- **Agenda.js, Nodemailer, Twilio:** Background job processing and multi-channel notification dispatch.
- **Zod & Helmet:** TypeScript-first schema validation and HTTP header security.

## 📂 Architecture & Project Structure

The project follows a standard decoupled Monorepo pattern with strict **3-Tier Clean Architecture** on the backend.

```text
Real-Time-Expert-Session-Booking-System/
├── backend/                  # Node.js, Express, MongoDB API
│   ├── src/
│   │   ├── config/           # Database and third-party configuration
│   │   ├── controllers/      # HTTP request parsing and response formatting
│   │   ├── middleware/       # Auth, Uploads, Error Handling, and Zod Validations
│   │   ├── models/           # Mongoose Schemas (User, Booking, Expert, etc.)
│   │   ├── repositories/     # Data Access Layer (DAL) for database interactions
│   │   ├── routes/           # API Endpoint mounts
│   │   ├── seeds/            # Database seeder scripts and utilities
│   │   ├── services/         # Core Business Logic and Transaction Boundaries
│   │   └── __tests__/        # Unit and Integration test suites
│   ├── .env.example          # Environment variable template
│   └── package.json          # Backend dependencies
├── frontend/                 # React + Vite Client
│   ├── src/
│   │   ├── components/       # Reusable UI components (Navbar, Cards, Modals)
│   │   ├── context/          # React Context providers (Auth, Notifications)
│   │   ├── pages/            # Routable application views (Dashboards, Booking, Chat)
│   │   └── services/         # Axios API interceptors and Socket connections
│   └── package.json          # Frontend dependencies
├── docs/                     # Engineering documentation and specifications
├── start.sh                  # Application startup utility script
└── README.md                 # Project documentation
```

## ⚙️ Getting Started

### Prerequisites
- **Node.js:** v18 or higher.
- **MongoDB:** A replica set configuration (for local standalone) or a MongoDB Atlas cluster URI (to support ACID transactions).
- **Redis:** (Optional for local development, required for multi-instance scaling).

### Quick Start
You can boot both the frontend and backend simultaneously using the provided auto-start script:
```bash
# Grants execution permissions and starts the dev servers
chmod +x start.sh
./start.sh
```
*(Proactively clears legacy ghost processes on ports 5000 and 5173 to ensure a clean startup).*

### Manual Setup

#### 1. Backend Setup
```bash
cd backend
npm install
```
Create a `.env` file in the `backend/` directory:
```env
PORT=5000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret_key
RAZORPAY_KEY_ID=your_razorpay_key
RAZORPAY_KEY_SECRET=your_razorpay_secret
RAZORPAY_WEBHOOK_SECRET=your_webhook_secret
```
*(Optional) Seed the database:*
```bash
node src/seeds/userSeeder.js
node src/seeds/expertSeeder.js
```
Start the backend server:
```bash
npm run dev
```

#### 2. Frontend Setup
```bash
cd frontend
npm install
npm run dev
```

## 📜 Coding Guidelines

- **Frontend:** Built on Vite. Use `npm run dev` for development, `npm run build` for production builds, and `npm run lint` for code quality checks.
- **Backend Architecture:** Adheres strictly to a **3-Tier Clean Architecture**. Responsibilities are separated across Routes, Controllers, Services (Domain Logic), Repositories (DAL), and Models. **Never** place business logic directly in the Controller.
- **Real-Time Communication:** Relies heavily on accurate Room management in Socket.io to optimize network bandwidth and ensure targeted broadcasts.
